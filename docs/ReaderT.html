ReaderT

ReaderT

new ReaderT()

Source:
  • reader/Reader.js, line 220

The monad transformer for the Reader type. This is useful for 'stacking' monads together to get more functionality.

Members

(static, constant) ask :ReaderT.<E, Monad.<A>>

Source:
  • reader/Reader.js, line 228
Since:
  • v1.0.2-beta

ReaderT.ask returns a static ReaderT instance that you can pass operations to.

Type:
  • ReaderT.<E, Monad.<A>>
Example
// ask :: Monad m => ReaderT e m a
import R from 'ramda';
import { Reader, Task } from 'zoomjs';

const RTask = Reader.T(Task);

RTask.ask.map(R.toUpper).run('foo'); // Task(null, foo)

(static) lift(monad) → {ReaderT.<E, Monad.<A>>}

Source:
  • reader/Reader.js, line 249
Since:
  • v1.0.2-beta

Lift an instance of the inner monad into a ReaderT and return it.

Example
// lift :: Monad m => m a -> ReaderT e m a
import { Reader, Task } from 'zoomjs';
import fs from 'fs';

const RTask = Reader.T(Task);

const readFile = Task.liftNode(fs.readFile);

ReaderT.ask
  .andThen(filepath =>
    RTask.lift(readFile(filepath)))
  .run('index.js');

// => Task(Error, Buffer)
Parameters:

The monad to lift into the ReaderT. It must be the same type of monad that was used to create the ReaderT.
